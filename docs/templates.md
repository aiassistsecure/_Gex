# 🧬 Jenny Template System

> **"A project is only as strong as its blueprint."**

Jenny uses a flexible, file-based template system to scaffold new applications. This document explains how templates are structured, how the variable injection works, and how to create your own custom blueprints.

---

## 🏗️ 1. Template Structure

All built-in templates are located in `python/jenny/templates/builtin/`. A valid template directory must follow this structure:

```text
my-template/
├── 📂 backend/         # Python source files
├── 📂 frontend/        # React source files
├── 📂 electron/        # Electron shell files
├── 📄 package.json     # Root Node dependencies
├── 📄 jenny.config.json # App configuration
└── 📄 template.json    # Template metadata (required)
```

### `template.json`
Every template requires a metadata file to describe itself to the Jenny CLI:

```json
{
    "id": "my-template",
    "name": "My Custom Template",
    "description": "A brief description of the stack",
    "version": "1.0.0",
    "stack": ["react", "fastapi"],
    "category": "full-stack"
}
```

## 🧪 2. The Variable System

When you run `jenny create <name>`, Jenny reads every text file in the template and performs a **Jinja2-style substitution**. This allow you to dynamically name classes, folders, and titles based on the user's project name.

### Available Variables:

| Variable | Description | Example (project "My App") |
|---|---|---|
| `{{ project_name }}` | Original name | `My App` |
| `{{ project_name_slug }}`| URL-safe slug | `my-app` |
| `{{ project_name_snake }}`| Python-safe snake case | `my_app` |
| `{{ project_name_pascal }}`| JS-safe PascalCase | `MyApp` |
| `{{ gene_version }}` | Framework version | `1.0.0` |

### Example Usage:
In a Python file (`app.py`):
```python
# Generated by Jenny
from fastapi import FastAPI
app = FastAPI(title="{{ project_name }}")
```

## 🛠️ 3. Available Blueprints

### `default`
The flagship Jenny blueprint. It provides a complete end-to-end stack:
- **Backend**: FastAPI with a health-monitor and basic routing ready.
- **Frontend**: React + Vite + TailwindCSS with a clean dark-mode shell.
- **Electron**: IPC-ready process manager that handles the backend lifecycle.

To use it:
```bash
jenny create my-app --template default
```

---

## 🎨 4. Creating Custom Templates

To create your own blueprint for your team or company:

1.  Navigate to `python/jenny/templates/builtin/`.
2.  Create a new folder (e.g., `modern-ai`).
3.  Copy the `template.json` from the `default` folder as a baseline.
4.  Add your source files, using `{{ variables }}` wherever you want dynamic naming.
5.  Test it by running: `jenny create test-proj --template modern-ai`.
