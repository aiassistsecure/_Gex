KeyStone-Lite is a free, open-source desktop application for AI-powered code debugging and editing. Built with Electron, it supports 11+ LLM providers and features surgical code editing, file context awareness, and three operational modes. This is a freemium lead-gen tool for the full AiAssist Secure platform.

B) Integrating with Your Codebase - top

KeyStone-Lite works with any existing codebase - no setup or configuration files needed. Here's how to get the most out of it:

Step 1: Open Your Project

Click Open Folder and select your project's root directory. Keystone will scan the folder structure automatically.

Step 2: Add Context Files

For best results, add the key files the AI should know about:

  • The file you're working on
  • Related files (imports, components, utilities)
  • Configuration files (package.json, tsconfig.json, etc.)

Use the + button in the sidebar, or type @filename in your message.

Step 3: Ask Questions or Request Changes

Example prompts to get started:

  • "Explain what this codebase does"
  • "Fix the bug in @UserAuth.tsx where login fails"
  • "Add input validation to the signup form"
  • "Refactor this function to be more readable"
  • "Write tests for the payment module"

C) Operational Modes - top

KeyStone-Lite offers three distinct operational modes, each designed for specific workflows:

Keystone Mode (Default)

Full-power code editing mode. The AI can read, analyze, and modify any file in your project. Supports surgical editing operations (REPLACE, INSERT, DELETE) that preserve your code structure.

Best for: Bug fixes, refactoring, implementing features, code review

Debug Mode

Issue analysis mode. The AI examines your code to identify bugs, trace errors, and explain what's going wrong. Helps you understand problems before fixing them.

Best for: Finding bugs, understanding errors, tracing issues, code analysis

Focus Mode

Research and documentation mode. The AI can read your codebase and search the web (via Tavily), but can only create/edit Markdown (.md) files. Perfect for documentation, research, and planning.

Best for: Writing docs, research, architecture planning, learning

Switch modes using the dropdown in the top navigation bar

D) Surgical Code Editing - top

Unlike traditional AI code editors that regenerate entire files, KeyStone-Lite uses surgical editing - precise, line-based operations that preserve your existing code structure.

Edit Operations

REPLACE - Replace specific lines with new content:

<<<EDIT path/to/file.ts>>>
[REPLACE lines 10-15]
// New code here
function updatedFunction() {
  return true;
}
<<</EDIT>>>

INSERT - Insert new lines at a specific position:

<<<EDIT path/to/file.ts>>>
[INSERT at line 25]
// This will be inserted before line 25
const newVariable = "hello";
<<</EDIT>>>

DELETE - Remove specific lines:

<<<EDIT path/to/file.ts>>>
[DELETE lines 50-55]
<<</EDIT>>>

CREATE - Create a new file:

<<<CREATE path/to/newfile.ts>>>
// File content here
export const myModule = {};
<<</CREATE>>>

Auto-Apply

Enable Auto-Apply in settings to automatically apply edits as they stream in. Disable it to review each edit before applying.

Warning: Surgical edits are based on line numbers. If you manually edit a file while Keystone is generating edits, line numbers may shift and cause incorrect placements.

E) File Context System - top

KeyStone-Lite maintains awareness of your project structure and can include relevant files in the context sent to the LLM.

Adding Context

  • Manual: Click the + button to add specific files
  • Auto-detect: Keystone automatically includes files mentioned in your prompts
  • @-mentions: Type @filename to reference a specific file

Context Panel

The left sidebar shows all files currently in context. You can:

  • Remove files by clicking the X
  • View file contents by clicking the file name
  • Clear all context with the Clear All button

Token Management

The context panel shows estimated token count. Stay within your model's context window for best results:

  • GPT-4 Turbo: 128K tokens
  • Claude 3: 200K tokens
  • Gemini 1.5: 1M tokens

F) Configuration - top

Settings Location

I) Troubleshooting - top

API Key Not Working

  • Verify the key is correct (no extra spaces)
  • Check that your API key has sufficient credits/quota
  • Ensure you've selected the correct provider

Edits Not Applying

  • Check that the file path exists in your project
  • Ensure the line numbers referenced still match the current file
  • Try enabling/disabling Auto-Apply

Slow Responses

  • Reduce context size by removing unused files
  • Try a faster model (e.g., GPT-4o-mini, Claude Haiku)
  • Check your internet connection

Logs Location

Debug logs are stored at:

  • Windows: %APPDATA%/keystone-lite/logs/
  • macOS: ~/Library/Logs/keystone-lite/
  • Linux: ~/.config/keystone-lite/logs/
